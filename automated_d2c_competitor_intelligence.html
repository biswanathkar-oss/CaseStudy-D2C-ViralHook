The Challenge: Platform Algorithmic Bias

Standard scraping outputs are fundamentally skewed. Reels and video formats dominate public metrics, completely hiding high-performing static strategies.

  • The Video View Skew: Passive video view counts range from 1,000 to 200,000+, while static image posts expose no public views.
  • Image Blindness: Standard sorting by engagement/view volume wipes out Carousel (Sidecar) posts entirely.
  • Token Financial Drain: Feeding raw, unfiltered scraper results straight to LLM vision models drains OpenAI API limits on junk content.

Step 2: Weighted Popularity Calculations

To find posts that genuinely resonated, the pipeline runs a weighted engagement formula via an inline JavaScript node.

Instead of counting a passive view equal to an active click, the formula scales values based on user intent and friction. A click to comment requires much more friction than a double-tap to like.

  • Likes are Weighted x5: Indicates deliberate interest.
  • Comments are Weighted x20: Proves deep brand dialogue.
// Weighted Popularity Algorithm (JS Node) for (let item of $input.all()) { const views = item.json.videoViewCount || 0; const likes = item.json.likesCount || 0; const comments = item.json.commentsCount || 0; // High friction engagement gets disproportionate weight item.json.popularityScore = views + (likes * 5) + (comments * 20); } let sorted = $input.all().sort((a, b) => b.json.popularityScore - a.json.popularityScore); return sorted.slice(0, 20); // Retain top 20 highest value anomalies

Step 3: Peering with the Content-Type Split

// Segmenting into Peer Swimlanes (JS Node) let videos = []; let images = []; for (let item of $input.all()) { if (item.json.VideoURL && item.json.VideoURL !== "null") { item.json.contentType = "video"; videos.push(item); } else { item.json.contentType = "image"; images.push(item); } } // Extract top 10 from each silo independently let topVideos = videos.slice(0, 10); let topImages = images.slice(0, 10); return [...topVideos, ...topImages]; // Balanced pool of 20 elements

Balancing Videos and Images

Sorting purely by raw popularity score would completely bury static image ads under viral video view counts. To prevent this, the second JavaScript node splits incoming assets into separate silos based on the presence of a video payload.

By evaluating the top 10 videos against videos, and the top 10 carousels against carousels, the downstream AI is guaranteed a perfectly balanced diet of visual strategies to analyze.

The Operational Result: Carousel, sidecar, and static photo strategies receive identical analytical attention as high-reach video Reels.

Feeding Live URLs to the Vision Model

Meta's CDN links contain temporary security tokens that expire within 24 hours. To bypass the slow, costly process of downloading binary images into n8n's memory, the pipeline dynamically extracts a flat imageUrl variable immediately after scraping.

By passing this active, live web link straight into the text prompt payload, gpt-4o-mini's vision model reads the image directly from Meta's servers, interpreting text overlays and creative layouts without any local storage friction.

Step 5: Enforcing Structured JSON Output

Standard LLM text outputs are impossible to parse into database rows. We use a JSON schema to enforce strict database columns.

The connected Structured Output Parser node acts as a hard physical mold. It forces the LLM's raw reasoning to output as a flat JSON dictionary containing exactly 6 keys, stripping away all conversational preamble.

  • Type Safety: Enforces numeric values for scores.
  • Zero Parsing Failures: Protects downstream spreadsheet nodes from breaking.
// The JSON Schema Enforced on GPT (Structured Parser) { "type": "object", "properties": { "post_description": { "type": "string" }, "targeted_user_persona": { "type": "string" }, "hook_type": { "type": "string" }, "hook_trigger_element": { "type": "string" }, "hook_score": { "type": "number" }, "hook_reason": { "type": "string" } }, "required": ["post_description", "targeted_user_persona", "hook_type", "hook_trigger_element", "hook_score", "hook_reason"] }
